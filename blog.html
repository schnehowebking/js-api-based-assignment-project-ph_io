Blog Questions With Answers

1. Discuss the scope of var, let, and const

Var: Function-scoped, hoisted to the top, can be redeclared and reassigned.
Let: Block-scoped, not hoisted, can be reassigned but not redeclared.
Const: Block-scoped, not hoisted, cannot be reassigned after declaration. For objects and arrays, the reference is constant, but their properties or elements can be modified.

2. Tell us the use cases of null and undefined

Undefined:

  • Default value for uninitialized variables and function parameters.
  • Indicates absence of a return value in functions.
  • Result of accessing properties or variables without assigned values.

Null:

  • Explicitly assigned to represent intentional absence of value.
  • Used to reset variables or indicate intentional emptiness.
  • A placeholder for intentional lack of an object value.

3. What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules for designing web services that follow the principles of REST. It uses standard HTTP methods, stateless communication, and resource-oriented URLs to enable interaction between software applications over the web.
